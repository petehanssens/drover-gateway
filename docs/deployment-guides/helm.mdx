## Deploy Bifrost with Helm

### 1. Prerequisites

- Kubernetes `v1.24+`
- `kubectl` configured
- Helm 3
- Provider API key(s)

### 2. Quick Start

Add repo:

```bash
helm repo add bifrost https://maximhq.github.io/bifrost/helm-charts
helm repo update
```

Prepare namespace and provider secret:

```bash
NAMESPACE="bifrost"
RELEASE_NAME="bifrost"

kubectl create namespace "${NAMESPACE}" --dry-run=client -o yaml | kubectl apply -f -

kubectl -n "${NAMESPACE}" create secret generic provider-keys \
  --from-literal=openai-api-key='sk-your-openai-key' \
  --dry-run=client -o yaml | kubectl apply -f -
```

Create `values-quickstart.yaml`:

```yaml
image:
  tag: "v1.5.0-prerelease5"

storage:
  mode: sqlite
  persistence:
    enabled: true
    size: 2Gi

bifrost:
  providers:
    openai:
      keys:
        - name: "openai-key"
          value: "env.OPENAI_API_KEY"
          weight: 1
          models: ["gpt-4o-mini"]
  providerSecrets:
    openai:
      existingSecret: "provider-keys"
      key: "openai-api-key"
      envVar: "OPENAI_API_KEY"
```

Install:

```bash
helm upgrade --install "${RELEASE_NAME}" bifrost/bifrost \
  --namespace "${NAMESPACE}" \
  -f values-quickstart.yaml \
  --wait --timeout 5m
```

What this does:
- Deploys Bifrost with SQLite persistence
- Configures OpenAI key via Kubernetes Secret
- Starts the API service on port `8080`

Access:

```bash
kubectl -n "${NAMESPACE}" port-forward svc/bifrost 8080:8080
curl -s http://localhost:8080/health
```

### 3. Basic Configuration (Optional)

```bash
helm upgrade "${RELEASE_NAME}" bifrost/bifrost \
  --namespace "${NAMESPACE}" \
  --reuse-values \
  --set 'bifrost.client.enableLogging=true'
```

### 4. Common Variations

<Tabs group="helm-variations">
<Tab title="Postgres + Providers">

```yaml
image:
  tag: "v1.5.0-prerelease5"

storage:
  mode: postgres

postgresql:
  enabled: true
  auth:
    username: "bifrost"
    password: "bifrost"
    database: "bifrost"
  primary:
    persistence:
      enabled: true
      size: 2Gi

bifrost:
  providers:
    openai:
      keys:
        - name: "openai-key"
          value: "env.OPENAI_API_KEY"
          weight: 1
          models: ["gpt-4o-mini"]
  providerSecrets:
    openai:
      existingSecret: "provider-keys"
      key: "openai-api-key"
      envVar: "OPENAI_API_KEY"
```

Meaning:
- `storage.mode: postgres` switches config/log stores to Postgres
- `postgresql.enabled: true` deploys chart-managed Postgres

</Tab>
<Tab title="Client Settings + Auth">

Create dashboard auth secret:

```bash
kubectl -n "${NAMESPACE}" create secret generic bifrost-admin-auth \
  --from-literal=BIFROST_ADMIN_USERNAME=admin \
  --from-literal=BIFROST_ADMIN_PASSWORD='change-me' \
  --dry-run=client -o yaml | kubectl apply -f -
```

```yaml
image:
  tag: "v1.5.0-prerelease5"

storage:
  mode: sqlite
  persistence:
    enabled: true
    size: 2Gi

bifrost:
  governance:
    authConfig:
      isEnabled: true
      disableAuthOnInference: false
      adminUsername: "env.BIFROST_ADMIN_USERNAME"
      adminPassword: "env.BIFROST_ADMIN_PASSWORD"
  client:
    enableLogging: true
    initialPoolSize: 301
    allowedOrigins: ["*", "https://app.example.com"]
    requiredHeaders: ["x-request-id"]
    loggingHeaders: ["x-request-id", "x-session-id"]
    headerFilterConfig:
      allowlist: ["x-trace-id"]
      denylist: ["authorization"]

envFrom:
  - secretRef:
      name: bifrost-admin-auth
```

Meaning:
- `governance.authConfig` enables admin auth for dashboard/API
- `client` tunes request handling, logging, headers, CORS

</Tab>
<Tab title="Governance (Teams)">

```yaml
image:
  tag: "v1.5.0-prerelease5"

storage:

    enabled: true

bifrost:
  governance:
    budgets:

    rateLimits:

        token_reset_duration: "1d"
        request_max_limit: 5000
        request_reset_duration: "1h"
    customers:
      - id: "customer-acme"
        name: "Acme Corp"
        budget_id: "budget-team-default"

    teams:
      - id: "team-platform"
        name: "Platform Team"
        customer_id: "customer-acme"
        budget_id: "budget-team-default"
        rate_limit_id: "rl-team-default"
```

Meaning:
- `budgets`/`rateLimits` define spend + request controls
- `customers`/`teams` assign policy scopes

<Tab title="Cluster Mode">

```yaml
bifrost:
  cluster:
    enabled: true
    peers: []
    gossip:
      port: 7946
      config:
        timeoutSeconds: 10
        successThreshold: 3
        failureThreshold: 3
    discovery:
      enabled: true
      type: kubernetes
      serviceName: "bifrost"
      k8sNamespace: "bifrost"
      k8sLabelSelector: "app.kubernetes.io/name=bifrost"
```

Meaning:
- `gossip` config controls cluster peer health checks
- `discovery` auto-registers peers from Kubernetes

## Helper reference (enum/dropdown fields)

- `storage.mode`: `sqlite` | `postgres`
- `vectorStore.type`: `none` | `weaviate` | `redis` | `qdrant` | `pinecone`
- `bifrost.cluster.discovery.type`: `kubernetes` | `dns` | `udp` | `consul` | `etcd` | `mdns`
- `bifrost.providers.<provider>.proxy_config.type`: `none` | `http` | `socks5` | `environment`

## Is this workable? (preflight checks)

```bash
# 1) Lint values against chart
helm lint bifrost/bifrost -f values-example.yaml

# 2) Render manifests
helm template "${RELEASE_NAME}" bifrost/bifrost \
  --namespace "${NAMESPACE}" \
  -f values-example.yaml > /tmp/bifrost-rendered.yaml

# 3) Client-side validation
kubectl apply --dry-run=client -f /tmp/bifrost-rendered.yaml

# 4) Server-side validation (optional)
kubectl apply --dry-run=server -f /tmp/bifrost-rendered.yaml
```

If all checks pass, the config is typically install-ready.

<Note>
`image.tag` is required in every values file.
</Note>

## Operations

### Upgrade

```bash
helm repo update

# Upgrade reusing all existing values
helm upgrade bifrost bifrost/bifrost --reuse-values

# Upgrade with new values
helm upgrade bifrost bifrost/bifrost -f your-values.yaml

# Upgrade and override a single field
helm upgrade bifrost bifrost/bifrost \
  --reuse-values \
  --set image.tag=v1.4.11
```

### Rollback

```bash
helm history bifrost
helm rollback bifrost          # to previous revision
helm rollback bifrost 2        # to specific revision
```

### Scale

```bash
kubectl scale deployment bifrost --replicas=5

# Or via Helm
helm upgrade bifrost bifrost/bifrost \
  --reuse-values \
  --set replicaCount=5
```

### Uninstall

```bash
helm uninstall bifrost

# Also remove PVCs (permanently deletes all data)
kubectl delete pvc -l app.kubernetes.io/instance=bifrost
```

## Monitoring

### Prometheus Metrics

Bifrost exposes Prometheus metrics at `/metrics`.

Enable ServiceMonitor for automatic scraping:

```yaml
serviceMonitor:
  enabled: true
  interval: 30s
  scrapeTimeout: 10s
```

### Health Checks

Check pod health:

```bash
# View pod status
kubectl get pods -l app.kubernetes.io/name=bifrost

# Check logs
kubectl logs -l app.kubernetes.io/name=bifrost --tail=100

# Describe pod
kubectl describe pod -l app.kubernetes.io/name=bifrost
```

### Metrics Endpoints

```bash
# Port forward
kubectl port-forward svc/bifrost 8080:8080

# Check metrics
curl http://localhost:8080/metrics

# Check health
curl http://localhost:8080/health
```

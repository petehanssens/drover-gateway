## Apply values

<Tabs group="values-apply-method">
<Tab title="One-liner (--set)">

Best for quick validation and temporary overrides.

```bash
helm install bifrost bifrost/bifrost \
  --set image.tag=v1.4.11 \
  --set replicaCount=3 \
  --set bifrost.client.initialPoolSize=500
```

</Tab>
<Tab title="Values file (-f)">

Best for team workflows and repeatable deployments.

```bash
# my-values.yaml
image:
  tag: "v1.4.11"

replicaCount: 2

bifrost:
  client:
    initialPoolSize: 500
    enableLogging: true
```

```bash
helm install bifrost bifrost/bifrost -f my-values.yaml
helm upgrade bifrost bifrost/bifrost -f my-values.yaml
```

</Tab>
<Tab title="Layered files">

Best for base + environment overlays.

```bash
helm install bifrost bifrost/bifrost \
  -f values/base.yaml \
  -f values/production.yaml
```

Later files override earlier files.

</Tab>
</Tabs>

## Object-by-object values reference

This section explains each top-level object one by one. For each object: what it controls, what fields expect, and what a sample config does.

<Tabs group="values-objects">
<Tab title="Bifrost">

`bifrost` is the main application object and maps into rendered `config.json`.

<Tabs group="bifrost-subobjects">
<Tab title="App + encryption + auth">

| Field | Meaning | Expects |
|---|---|---|
| `bifrost.appDir` | On-container data/config path used by runtime. | Absolute Linux path. |
| `bifrost.port` | Bifrost process listening port inside container. | Integer port. |
| `bifrost.host` | Bind address for Bifrost process. | IP/host string, commonly `0.0.0.0`. |
| `bifrost.logLevel` | Runtime logging verbosity. | Supported level string (`info`, `debug`, etc.). |
| `bifrost.logStyle` | Log output format style. | Style string (for example `json`). |
| `bifrost.encryptionKey` | Inline encryption key for sensitive fields. | String or `env.VAR` reference. |
| `bifrost.encryptionKeySecret.name/key` | Secret source for encryption key env injection. | Secret name + key string. |
| `bifrost.governance.authConfig.*` | Current admin authentication settings for dashboard/API access. | Admin auth fields (`adminUsername`, `adminPassword`, toggles, secret refs). |

```yaml
bifrost:
  appDir: /app/data
  port: 8080
  host: 0.0.0.0
  logLevel: info
  logStyle: json
  encryptionKeySecret:
    name: bifrost-encryption
    key: encryption-key
  governance:
    authConfig:
      isEnabled: true
      existingSecret: bifrost-admin-auth
      usernameKey: username
      passwordKey: password
```

This sample secures encryption/admin credentials using Kubernetes secrets and runs Bifrost on standard bind settings.

</Tab>
<Tab title="Client">

`bifrost.client` configures request handling, logging, CORS, headers, and compatibility behavior.

| Field | Meaning | Expects |
|---|---|---|
| `dropExcessRequests` | Drops requests instead of queueing beyond capacity. | Boolean. |
| `initialPoolSize` | Worker pool pre-allocation per provider queue. | Integer > 0. |
| `allowedOrigins[]` | CORS allowed origins for browser calls. | Origin strings (`*` or specific URLs). |
| `enableLogging` | Enables request/response log persistence. | Boolean. |
| `disableContentLogging` | Redacts request/response body content from logs. | Boolean. |
| `logRetentionDays` | Log retention period for store cleanup. | Integer days. |
| `allowDirectKeys` | Allows incoming raw provider keys in requests. | Boolean. |
| `maxRequestBodySizeMb` | Max request body size accepted by API. | Integer MB. |
| `headerFilterConfig.allowlist/denylist` | Filters forwarded extra headers (`x-bf-eh-*`). | Header-name arrays. |
| `compat.*` | Parameter conversion/drop behavior for SDK compatibility. | Boolean flags. |

```yaml
bifrost:
  client:
    initialPoolSize: 500
    enableLogging: true
    disableContentLogging: false
    allowedOrigins:
      - "https://app.example.com"
    requiredHeaders:
      - "x-request-id"
    loggingHeaders:
      - "x-request-id"
    headerFilterConfig:
      allowlist: ["x-trace-id"]
      denylist: ["authorization"]
```

This sample enables operational logging and strict header policy while preserving required request metadata.

</Tab>
<Tab title="Providers + providerSecrets">

| Field | Meaning | Expects |
|---|---|---|
| `bifrost.providers` | Full provider block passed to runtime config. | Provider-keyed object matching schema. |
| `bifrost.providers.<provider>.keys[]` | API key records and traffic weights. | Objects with `name`, `value`, `weight` and optional model filters. |
| `bifrost.providers.<provider>.network_config` | HTTP timeout/retry/header tuning per provider. | Provider network config object. |
| `bifrost.providers.<provider>.proxy_config` | Proxy routing and auth for provider calls. | Proxy config (`type`, `url`, credentials). |
| `bifrost.providerSecrets.<provider>.existingSecret` | Secret containing provider API key. | Secret name string. |
| `bifrost.providerSecrets.<provider>.key` | Secret key for actual provider key value. | Secret data key string. |
| `bifrost.providerSecrets.<provider>.envVar` | Env var name injected into container for provider key resolution. | Env var string (`OPENAI_API_KEY`). |

```yaml
bifrost:
  providers:
    openai:
      keys:
        - name: "openai-primary"
          value: "env.OPENAI_API_KEY"
          weight: 1
  providerSecrets:
    openai:
      existingSecret: "provider-keys"
      key: "openai-api-key"
      envVar: "OPENAI_API_KEY"
```

This sample injects the OpenAI key via Kubernetes secret and references it safely in provider configuration.

</Tab>
<Tab title="MCP + plugins + governance">

| Field | Meaning | Expects |
|---|---|---|
| `bifrost.mcp.enabled` | Enables MCP client/tool subsystem. | Boolean. |
| `bifrost.mcp.clientConfigs[]` | MCP client definitions (stdio/http/streamable). | Client config objects. |
| `bifrost.mcp.toolManagerConfig.*` | Tool execution timeout and agent recursion depth. | Integer config values. |
| `bifrost.plugins.<name>.enabled` | Enables built-in plugin. | Boolean. |
| `bifrost.plugins.<name>.version` | Plugin config version for upgrade semantics. | Integer >= 1. |
| `bifrost.plugins.<name>.config` | Plugin-specific operational settings. | Object matching plugin schema. |
| `bifrost.governance.budgets[]` | Seed spend limits. | Budget objects (`id`, `max_limit`, `reset_duration`). |
| `bifrost.governance.rateLimits[]` | Seed token/request limits. | Rate-limit objects with reset durations. |
| `bifrost.governance.virtualKeys[]` | Seed virtual keys and provider/model policies. | Virtual-key objects. |
| `bifrost.governance.routingRules[]` | CEL routing policies and weighted targets. | Routing rule objects. |

```yaml
bifrost:
  mcp:
    enabled: true
    toolManagerConfig:
      toolExecutionTimeout: 30
      maxAgentDepth: 10
  plugins:
    semanticCache:
      enabled: true
      version: 1
      config:
        provider: "openai"
        embedding_model: "text-embedding-3-small"
        dimension: 1536
        threshold: 0.85
        ttl: "1h"
  governance:
    budgets:
      - id: "default-budget"
        max_limit: 200
        reset_duration: "1M"
```

This sample enables tool-calling via MCP, semantic cache behavior, and a monthly governance spend cap.

</Tab>
<Tab title="Enterprise objects">

| Field | Meaning | Expects |
|---|---|---|
| `bifrost.cluster.*` | Multi-node cluster and peer-discovery behavior. | Cluster config object (gossip/discovery/peers). |
| `bifrost.scim.*` | Enterprise identity provisioning and SSO config. | Provider-specific SCIM object (`okta`/`entra`). |
| `bifrost.loadBalancer.*` | Adaptive routing/load balancer internals. | Tracker/bootstrap object values. |
| `bifrost.guardrails.*` | Content safety providers and rules. | Rules/providers arrays. |
| `bifrost.accessProfiles[]` | RBAC access profile templates with provider/model/tool policies. | Access profile array objects. |
| `bifrost.auditLogs.*` | Enterprise CADF-style audit logging controls. | `disabled` flag and HMAC key config. |

```yaml
bifrost:
  cluster:
    enabled: true
    gossip:
      port: 7946
    discovery:
      enabled: true
      type: kubernetes
      serviceName: bifrost
      k8sNamespace: bifrost
  auditLogs:
    disabled: false
    hmacKey: "env.BIFROST_AUDIT_HMAC_KEY"
```

This sample enables cluster peer discovery in Kubernetes and signs audit log events with an HMAC key.

<Tab title="Core k8s objects">

### `image`

Controls which container image is deployed.

| Field | Meaning | Expects |
|---|---|---|
| `image.repository` | Image registry path used by Kubernetes pull operations. | Fully qualified image repository string. |
| `image.tag` | Runtime version for Bifrost binary. Required by chart validation. | Version string like `v1.5.0` or prerelease tag. |
| `image.pullPolicy` | Pull strategy for container updates. | `IfNotPresent`, `Always`, or `Never`. |
| `imagePullSecrets[]` | Credentials for private container registries. | Array of secret refs with `name`. |

```yaml
image:
  repository: docker.io/maximhq/bifrost
  tag: "v1.5.0-prerelease5"
  pullPolicy: IfNotPresent
```

This sample pins Bifrost to a known version and avoids unnecessary image repulls.

### `replicaCount`, `resources`, probes, lifecycle

Control pod count, sizing, and health behavior.

| Field | Meaning | Expects |
|---|---|---|
| `replicaCount` | Fixed pod count when autoscaling is disabled. | Integer >= 1. |
| `resources.requests.*` | Guaranteed CPU/memory reserved by scheduler. | Kubernetes resource quantities (`500m`, `512Mi`). |
| `resources.limits.*` | Hard max CPU/memory allowed per pod. | Kubernetes resource quantities (`2Gi`, `2000m`). |
| `livenessProbe` | Restart condition if process becomes unhealthy. | Valid probe object (httpGet/exec/tcpSocket + timings). |
| `readinessProbe` | Traffic gating condition for healthy pods. | Valid probe object. |
| `terminationGracePeriodSeconds` | Shutdown drain window before force kill. | Integer seconds. |
| `lifecycle.preStop.exec.command` | Pre-shutdown command to drain in-flight traffic. | Command array (for example shell sleep). |

```yaml
replicaCount: 2
resources:
  requests:
    cpu: 500m
    memory: 512Mi
  limits:
    cpu: 2000m
    memory: 2Gi
terminationGracePeriodSeconds: 60
lifecycle:
  preStop:
    exec:
      command: ["sh", "-c", "sleep 15"]
```

This sample keeps two pods alive, caps resource usage, and gives streaming requests time to finish on rollout/scale-down.

</Tab>
<Tab title="Service, ingress, scaling">

### `service`

Defines internal service exposure in-cluster.

| Field | Meaning | Expects |
|---|---|---|
| `service.type` | Kubernetes service exposure mode. | `ClusterIP`, `NodePort`, or `LoadBalancer`. |
| `service.port` | Port exposed by service and used by ingress backend. | Integer TCP port. |
| `service.annotations` | Cloud/controller specific service behavior. | Key/value annotation map. |

### `ingress`

Defines external HTTP(S) entrypoint.

| Field | Meaning | Expects |
|---|---|---|
| `ingress.enabled` | Whether to create Ingress resource. | Boolean. |
| `ingress.className` | Which ingress controller should process this ingress. | Class name string (for example `nginx`). |
| `ingress.annotations` | TLS, cert-manager, proxy tuning, auth annotations. | Key/value map. |
| `ingress.hosts[]` | Host and path routing rules. | Array of host/path objects. |
| `ingress.tls[]` | TLS secret-host bindings. | Array with `secretName` and host list. |

### `autoscaling`

Defines HPA behavior.

| Field | Meaning | Expects |
|---|---|---|
| `autoscaling.enabled` | Switches from fixed replicas to HPA-managed replicas. | Boolean. |
| `autoscaling.minReplicas` | Minimum pods maintained under low load. | Integer >= 1. |
| `autoscaling.maxReplicas` | Max pods created under high load. | Integer >= `minReplicas`. |
| `autoscaling.targetCPUUtilizationPercentage` | CPU threshold for scaling decisions. | Integer percent. |
| `autoscaling.targetMemoryUtilizationPercentage` | Memory threshold for scaling decisions. | Integer percent. |
| `autoscaling.behavior` | Scale-up/scale-down smoothing and policy limits. | Valid HPA behavior object. |

```yaml
service:
  type: ClusterIP
  port: 8080
ingress:
  enabled: true
  className: nginx
  hosts:
    - host: bifrost.example.com
      paths:
        - path: /
          pathType: Prefix
autoscaling:
  enabled: true
  minReplicas: 2
  maxReplicas: 10
  targetCPUUtilizationPercentage: 80
```

This sample exposes Bifrost at `bifrost.example.com` and allows HPA to scale pods from 2 to 10 with CPU-driven scaling.

</Tab>
<Tab title="Storage objects">

### `storage`

Global store selection and persistence behavior.

| Field | Meaning | Expects |
|---|---|---|
| `storage.mode` | Default database backend for config/log stores. | `sqlite` or `postgres`. |
| `storage.persistence.*` | PVC config for local persistent data (especially SQLite). | Valid persistence object (size, accessMode, class). |
| `storage.configStore.type` | Store type override for config store. | Empty, `sqlite`, or `postgres`. |
| `storage.logsStore.type` | Store type override for logs store. | Empty, `sqlite`, or `postgres`. |
| `storage.logsStore.objectStorage.*` | Offloads large logs payloads to object storage. | Object-storage config (S3/GCS). |

### `postgresql`

Controls embedded or external PostgreSQL.

| Field | Meaning | Expects |
|---|---|---|
| `postgresql.enabled` | Deploy bundled Postgres subchart. | Boolean. |
| `postgresql.external.enabled` | Use external Postgres instead of subchart. | Boolean. |
| `postgresql.external.host/port/user/database` | External DB connectivity parameters. | Valid host/port/user/database values. |
| `postgresql.external.password` | External DB password inline (prefer secret). | String. |
| `postgresql.external.existingSecret/passwordKey` | Secret-based DB password injection. | Secret name + key path. |

### `vectorStore`

Controls semantic-cache backend connectivity.

| Field | Meaning | Expects |
|---|---|---|
| `vectorStore.enabled` | Enables vector-store support in rendered config. | Boolean. |
| `vectorStore.type` | Selected vector backend. | `none`, `weaviate`, `redis`, `qdrant`, `pinecone`. |
| `vectorStore.weaviate.*` | In-cluster or external Weaviate settings. | Valid weaviate sub-object. |
| `vectorStore.redis.*` | In-cluster or external Redis/Valkey settings. | Valid redis sub-object. |
| `vectorStore.qdrant.*` | In-cluster or external Qdrant settings. | Valid qdrant sub-object. |
| `vectorStore.pinecone.external.*` | Pinecone external connection settings. | API key + index host (secret preferred). |

```yaml
storage:
  mode: postgres
postgresql:
  enabled: false
  external:
    enabled: true
    host: "my-postgres.example.com"
    port: 5432
    user: "bifrost"
    database: "bifrost"
    existingSecret: "postgres-credentials"
    passwordKey: "password"
vectorStore:
  enabled: true
  type: redis
  redis:
    external:
      enabled: true
      host: "redis.example.com"
      port: 6379
      existingSecret: "redis-credentials"
      passwordKey: "password"
```

This sample runs Bifrost with external Postgres for config/log data and external Redis for vector operations.

</Tab>
<Tab title="Runtime env + pod scheduling">

### Environment injection objects

| Field | Meaning | Expects |
|---|---|---|
| `env[]` | Explicit env var entries in pod spec. | Array of `name` + `value/valueFrom`. |
| `extraEnv` | Map-style env vars appended after `env[]`. | Key/value object. |
| `envFrom[]` | Bulk env import from Secret/ConfigMap refs. | Array of `secretRef`/`configMapRef`. |

### Pod identity + scheduling objects

| Field | Meaning | Expects |
|---|---|---|
| `serviceAccount.*` | Pod identity object (create/name/annotations/automount). | Service account config object. |
| `podSecurityContext` | Pod-level security defaults (UID/fsGroup/runAsNonRoot). | Valid security context object. |
| `securityContext` | Container-level security hardening. | Valid container security context object. |
| `nodeSelector` | Hard scheduling constraints by node labels. | Label map. |
| `tolerations[]` | Allows scheduling onto tainted nodes. | Array of toleration objects. |
| `affinity` | Affinity/anti-affinity scheduling rules. | Kubernetes affinity object. |
| `volumes[]` | Additional pod volumes. | Kubernetes volume object array. |
| `volumeMounts[]` | Additional container mounts for extra volumes. | Kubernetes volumeMount array. |
| `initContainers[]` | Pre-start bootstrap containers (migrations/checks/waits). | Kubernetes container object array. |
| `deploymentAnnotations`, `deploymentLabels`, `podAnnotations`, `podLabels` | Metadata hooks for ops tooling/observability. | Key/value maps. |

```yaml
serviceAccount:
  create: true
  annotations:
    eks.amazonaws.com/role-arn: "arn:aws:iam::123456789012:role/bifrost-role"
nodeSelector:
  node-type: ai-workload
tolerations:
  - key: "gpu"
    operator: "Equal"
    value: "true"
    effect: "NoSchedule"
affinity:
  podAntiAffinity:
    requiredDuringSchedulingIgnoredDuringExecution:
      - labelSelector:
          matchLabels:
            app.kubernetes.io/name: bifrost
        topologyKey: kubernetes.io/hostname
```

This sample enforces secure pod identity and spreads Bifrost replicas across hosts while allowing GPU node scheduling.

## Helm Operations

### View current values

```bash
helm get values bifrost
```

### Diff before upgrading (requires helm-diff plugin)

```bash
helm diff upgrade bifrost bifrost/bifrost -f my-values.yaml
```

### Rollback

```bash
helm history bifrost
helm rollback bifrost       # to previous revision
helm rollback bifrost 2     # to revision 2
```

### Uninstall

```bash
helm uninstall bifrost

# Also remove PVCs (deletes all data)
kubectl delete pvc -l app.kubernetes.io/instance=bifrost
```

## Advanced Configuration

### Comprehensive Example

A production-ready values file combining the most common settings:

```yaml
# my-values.yaml
image:
  tag: "v1.4.11"

replicaCount: 3

storage:
  mode: postgres

postgresql:
  enabled: true
  auth:
    password: "secure-password"   # use existingSecret in production

autoscaling:
  enabled: true
  minReplicas: 3
  maxReplicas: 10

ingress:
  enabled: true
  className: nginx
  hosts:
    - host: bifrost.example.com
      paths:
        - path: /
          pathType: Prefix

bifrost:
  encryptionKeySecret:
    name: "bifrost-encryption"
    key: "key"
  providers:
    openai:
      keys:
        - name: "primary"
          value: "env.OPENAI_API_KEY"
          weight: 1
  providerSecrets:
    openai:
      existingSecret: "provider-api-keys"
      key: "openai-api-key"
      envVar: "OPENAI_API_KEY"
```

```bash
helm install bifrost bifrost/bifrost -f my-values.yaml
```

### Node Affinity & Scheduling

Deploy to specific nodes and spread replicas across hosts:

```yaml
nodeSelector:
  node-type: ai-workload

affinity:
  podAntiAffinity:
    requiredDuringSchedulingIgnoredDuringExecution:
      - labelSelector:
          matchLabels:
            app.kubernetes.io/name: bifrost
        topologyKey: kubernetes.io/hostname

tolerations:
  - key: "gpu"
    operator: "Equal"
    value: "true"
    effect: "NoSchedule"
```

### Deployment & Pod Annotations

Useful for tooling like [Keel](https://keel.sh) for automatic image updates or Datadog APM injection:

```yaml
deploymentAnnotations:
  keel.sh/policy: force
  keel.sh/trigger: poll

podAnnotations:
  ad.datadoghq.com/bifrost.logs: '[{"source":"bifrost","service":"bifrost"}]'
```

## Common Patterns

Ready-made values files for the most common deployment scenarios. Each pattern builds on the [quickstart](/deployment-guides/helm).

<Tabs>
<Tab title="Development">

Simple setup for local testing. SQLite, single replica, no autoscaling.

```bash
helm install bifrost bifrost/bifrost \
  --set image.tag=v1.4.11 \
  --set 'bifrost.providers.openai.keys[0].name=dev-key' \
  --set 'bifrost.providers.openai.keys[0].value=sk-your-key' \
  --set 'bifrost.providers.openai.keys[0].weight=1'
```

```bash
# Access
kubectl port-forward svc/bifrost 8080:8080
```

</Tab>
<Tab title="Multi-Provider">

Multiple LLM providers with weighted load balancing.

```bash
kubectl create secret generic provider-keys \
  --from-literal=openai-api-key='sk-...' \
  --from-literal=anthropic-api-key='sk-ant-...' \
  --from-literal=gemini-api-key='your-gemini-key'
```

```yaml
# multi-provider.yaml
image:
  tag: "v1.4.11"

bifrost:
  encryptionKey: "your-encryption-key"

  client:
    enableLogging: true
    allowDirectKeys: false

  providers:
    openai:
      keys:
        - name: "openai-primary"
          value: "env.OPENAI_API_KEY"
          weight: 2    # 50% of traffic
    anthropic:
      keys:
        - name: "anthropic-primary"
          value: "env.ANTHROPIC_API_KEY"
          weight: 1    # 25%
    gemini:
      keys:
        - name: "gemini-primary"
          value: "env.GEMINI_API_KEY"
          weight: 1    # 25%

  providerSecrets:
    openai:
      existingSecret: "provider-keys"
      key: "openai-api-key"
      envVar: "OPENAI_API_KEY"
    anthropic:
      existingSecret: "provider-keys"
      key: "anthropic-api-key"
      envVar: "ANTHROPIC_API_KEY"
    gemini:
      existingSecret: "provider-keys"
      key: "gemini-api-key"
      envVar: "GEMINI_API_KEY"

  plugins:
    telemetry:
      enabled: true
    logging:
      enabled: true
```

```bash
helm install bifrost bifrost/bifrost -f multi-provider.yaml
```

```bash
kubectl create secret generic postgres-credentials \
  --from-literal=password='your-external-postgres-password'
```

```yaml
# external-db.yaml
image:
  tag: "v1.4.11"

storage:
  mode: postgres

  enabled: false

    user: "bifrost"

    sslMode: "require"
    existingSecret: "postgres-credentials"
    passwordKey: "password"

bifrost:
  encryptionKey: "your-encryption-key"

  providers:
    openai:
      keys:
        - name: "openai-primary"
          value: "sk-..."
          weight: 1
```

```bash
helm install bifrost bifrost/bifrost -f external-db.yaml
```

<Tab title="AI Workloads">

Semantic response caching for high-volume AI inference.

```bash
kubectl create secret generic bifrost-encryption \
  --from-literal=key='your-32-byte-encryption-key'

kubectl create secret generic provider-keys \
  --from-literal=openai-api-key='sk-your-key'
```

```yaml
# ai-workload.yaml
image:
  tag: "v1.4.11"

storage:
  mode: postgres

postgresql:
  enabled: true
  auth:
    password: "secure-password"
  primary:
    persistence:
      size: 50Gi

vectorStore:
  enabled: true
  type: weaviate
  weaviate:
    enabled: true
    persistence:
      size: 50Gi

bifrost:
  encryptionKeySecret:
    name: "bifrost-encryption"
    key: "key"

  providers:
    openai:
      keys:
        - name: "openai-primary"
          value: "env.OPENAI_API_KEY"
          weight: 1

  providerSecrets:
    openai:
      existingSecret: "provider-keys"
      key: "openai-api-key"
      envVar: "OPENAI_API_KEY"

  plugins:
    semanticCache:
      enabled: true
      config:
        provider: "openai"
        embedding_model: "text-embedding-3-small"
        dimension: 1536
        threshold: 0.85
        ttl: "1h"
        cache_by_model: true
        cache_by_provider: true
```

```bash
helm install bifrost bifrost/bifrost -f ai-workload.yaml
```

<Tab title="Kubernetes Secrets Only">

Zero credentials in values files — all sensitive data in Kubernetes Secrets.

```bash
kubectl create secret generic postgres-credentials \
  --from-literal=password='your-postgres-password'

kubectl create secret generic bifrost-encryption \
  --from-literal=key='your-encryption-key'

kubectl create secret generic provider-keys \
  --from-literal=openai-api-key='sk-...' \
  --from-literal=anthropic-api-key='sk-ant-...'

kubectl create secret generic qdrant-credentials \
  --from-literal=api-key='your-qdrant-api-key'
```

```yaml
# secrets-only.yaml
image:
  tag: "v1.4.11"

storage:
  mode: postgres

postgresql:
  enabled: false
  external:
    enabled: true
    host: "postgres.example.com"
    port: 5432
    user: "bifrost"
    database: "bifrost"
    sslMode: "require"
    existingSecret: "postgres-credentials"
    passwordKey: "password"

vectorStore:
  enabled: true
  type: qdrant
  qdrant:
    enabled: false
    external:
      enabled: true
      host: "qdrant.example.com"
      port: 6334
      existingSecret: "qdrant-credentials"
      apiKeyKey: "api-key"

bifrost:
  encryptionKeySecret:
    name: "bifrost-encryption"
    key: "key"

  providers:
    openai:
      keys:
        - name: "openai-primary"
          value: "env.OPENAI_API_KEY"
          weight: 1
    anthropic:
      keys:
        - name: "anthropic-primary"
          value: "env.ANTHROPIC_API_KEY"
          weight: 1

  providerSecrets:
    openai:
      existingSecret: "provider-keys"
      key: "openai-api-key"
      envVar: "OPENAI_API_KEY"
    anthropic:
      existingSecret: "provider-keys"
      key: "anthropic-api-key"
      envVar: "ANTHROPIC_API_KEY"
```

```bash
helm install bifrost bifrost/bifrost -f secrets-only.yaml
```
